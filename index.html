Local Stories

A markup library for making dynamic, localised stories

Demo introduction

Hello! This is the beta demo page for writers wanting to use the library to write some amazing localised stories. Thank you for showing interest. Hopefully this page will help you work out how the library works (and how it doesn't as it's still in development and no doubt buggy).

The library is made to be very straightforward to write into text and hopefully won't get in the way of your good story. Here's how it works.

  • You write a story
  • You enclose place types (aquarium, train station, Japanese restaurant...) in square brackets and asign each one an ID number rather. These are called variables.
  • You put in other types of variables such as the date and time of the user.
  • You decide that certain bits of the story should only take place if say, it is a Tuesday and it's snowing, in June and the reader is in Newcastle. You enclose these parts in a final set of variables, conditionals.
  • The system detects a reader's location and automatically replaces these variables with dynamic, localised information.
  • The story is then unique to every reader but keeps its essense.

Types of variable (syntax)

Places

Places are pulled in from Foursquare. A place-based social network and place listing.

To put a place in your story, a zoo for example, first look up the name on this list from Foursquare to make sure it's available. The list is updated live from Foursquare so if squirrel cafes suddenly become a huge thing they might be a useable variable in the future.

Next, enclose the name of the category in square brackets followed by a | (pipe) character and a unique identifier for that place (in case you want to re-use the same place in multiple parts of the story).

Example: [zoo|1]
Every instance of [zoo|1] would be replaced by the nearest zoo to a reader automatically. [zoo|2] would be another zoo.

Other variables

It's not just places you can swap into text, here are some other variables you can use. Unlike places, you don't need to put in the | and an ID number. These always print out the same result for a reader.

  • [dayofweek] - prints out Monday, Tuesday... etc
  • [hours24] - Hour in day in 24 hour time. So 13 for 1pm
  • [hours12] - Hour in day in 12 hour time. So 1 for 1pm
  • [minutes] - Minutes past the hour
